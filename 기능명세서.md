# BOOK TO LEARN & RUN 기능 명세서

## 1. 문서 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | BOOK TO LEARN & RUN (BTLR) |
| 서비스 주소 | https://btlr.vercel.app |
| 서비스 목적 | 도서 검색, 대출, 예약 및 개인 독서 현황 관리를 제공하는 웹 기반 도서관 프로젝트 |
| 주요 사용자 | 비회원, 일반 회원, 관리자 |
| 프런트엔드 | HTML, CSS, JavaScript |
| 백엔드 | Supabase Authentication, Database, Storage |
| AI 기능 | Gemini API 기반 도서 정보 자동 작성 |
| 배포 환경 | Vercel |

## 2. 사용자 권한

| 구분 | 이용 가능 기능 |
|---|---|
| 비회원 | 홈 열람, 도서 검색·필터·정렬, 도서 상세 열람, 회원가입, 로그인 |
| 일반 회원 | 비회원 기능, 도서 찜, 대출, 반납, 예약, 예약 취소, 마이페이지, 이름·비밀번호 변경 |
| 관리자 | 도서 및 회원 정보 조회, 회원 수정·삭제, 도서 추가·수정·삭제, 표지 업로드, AI 자동입력 |

관리자 계정은 서비스 운영용 계정이므로 도서 대출과 예약을 할 수 없다. 관리자 전용 기능은 화면뿐 아니라 Supabase 함수와 데이터베이스 정책에서도 권한을 확인한다.

## 3. 화면 구성

| 화면 ID | 화면명 | 파일 | 주요 기능 |
|---|---|---|---|
| P-01 | 홈 | `index.html` | 통합 검색, 추천 도서, 관심사 탐색, 최근 검색어, 대출 가능 도서 수 표시 |
| P-02 | 도서 검색 | `search.html` | 검색 결과, 카테고리·상태 필터, 정렬, 도서 카드 표시 |
| P-03 | 도서 상세 | `detail.html` | 상세 정보, 재고, 찜, 대출·예약 진입, 관련 도서 표시 |
| P-04 | 예약 신청 | `reserve.html` | 예약 조건 확인, 예약 신청 및 완료 안내 |
| P-05 | 회원가입 | `signup.html` | 아이디, 이름, 이메일, 비밀번호로 계정 생성 |
| P-06 | 로그인 | `login.html` | 아이디 또는 이메일 기반 로그인 |
| P-07 | 마이페이지 | `mypage.html` | 회원정보 수정, 찜·대출·예약 목록 관리 |
| P-08 | 관리자 | `admin.html` | 회원 관리, 도서 관리, AI 도서 정보 생성 |

### 4.3 회원 인증 및 회원정보

| AUTH-02 | 아이디 검증 | 비회원 | 아이디를 소문자로 정규화하고 형식을 검사한다. | 영문 소문자로 시작하며 영문·숫자·밑줄 조합 4~20자만 허용한다. |
| AUTH-03 | 이름 검증 | 비회원 | 앞뒤 공백과 연속 공백을 정리해 저장한다. | 1~30자이며 정규화된 이름의 중복을 허용하지 않는다. |
| AUTH-04 | 이메일 검증 | 비회원 | 이메일을 소문자로 정규화하고 이메일 형식을 검사한다. | Supabase Auth에서 중복 이메일 생성을 차단한다. |
| AUTH-05 | 비밀번호 검증 | 비회원 | Supabase Auth에 비밀번호를 전달해 안전하게 계정을 생성한다. | 최소 8자이며 원문 비밀번호는 서비스 DB나 관리자 화면에 저장·표시하지 않는다. |
| AUTH-06 | 로그인 | 비회원 | 아이디 또는 이메일과 비밀번호로 로그인한다. 아이디 입력 시 연결된 이메일을 조회한 뒤 인증한다. | 잘못된 정보 입력 시 구체적인 계정 존재 여부를 노출하지 않는 오류 메시지를 표시한다. |
| AUTH-07 | 세션 유지 | 회원·관리자 | Supabase 세션을 기반으로 새로고침 후에도 로그인 상태를 복원한다. | 세션이 만료되거나 유효하지 않으면 비로그인 상태로 처리한다. |
| USER-01 | 이름 변경 | 회원·관리자 | 마이페이지에서 본인의 표시 이름을 변경한다. | 공백 정규화 후 1~30자, 중복 이름 불가 조건을 적용한다. |

### 4.4 찜·대출·예약 및 마이페이지

|---|---|---|---|---|

| CIR-03 | 반납 예정일 | 일반 회원 | 대출 시점으로부터 14일 뒤를 기본 반납 예정일로 생성한다. | 서버 시간을 기준으로 생성한다. |

| CIR-05 | 재고·상태 자동 동기화 | 전체 | 가용 수량이 1 이상이면 `대출 가능`, 0이면 `대출 중`으로 표시한다. | 가용 수량은 0 미만 또는 전체 수량 초과가 될 수 없다. |

| CIR-07 | 예약 취소 | 일반 회원 | 마이페이지에서 본인의 예약을 취소한다. | 본인의 예약만 취소할 수 있다. |
| MY-01 | 회원 요약 | 회원·관리자 | 이름, 이메일, 가입일, 권한을 표시한다. | 로그인한 사용자만 접근할 수 있다. |
| MY-02 | 찜 목록 | 일반 회원 | 찜한 도서 목록과 건수를 표시하고 찜 취소 기능을 제공한다. | - |
| MY-03 | 대출 목록 | 일반 회원 | 현재 대출 중인 도서, 반납 예정일과 건수를 표시하고 반납 기능을 제공한다. | 반납 완료 기록은 현재 목록에서 제외한다. |
| MY-04 | 예약 목록 | 일반 회원 | 예약한 도서 목록과 건수를 표시하고 예약 취소 기능을 제공한다. | - |

### 4.5 관리자 기능

| 기능 ID | 기능명 | 대상 | 처리 내용 | 예외·제한 |
|---|---|---|---|---|
| ADM-01 | 관리자 페이지 접근 | 관리자 | 로그인한 사용자의 프로필 권한이 `admin`인지 확인한 후 관리자 화면을 표시한다. | 비로그인 사용자는 로그인 화면으로 이동하고 일반 회원은 접근을 거부한다. |
| ADM-02 | 회원 목록 조회 | 관리자 | 아이디, 이름, 이메일, 권한, 가입일, 최근 로그인 정보를 조회한다. | 관리자 계정은 목록에서 제외하며 비밀번호는 조회하지 않는다. |
| ADM-03 | 회원 정보 통합 저장 | 관리자 | 한 행의 이름과 권한 변경을 하나의 저장 버튼으로 처리한다. | 이름 검증과 권한 값 검증을 서버에서 다시 수행한다. |
| ADM-04 | 회원 삭제 | 관리자 | 확인창 승인 후 프로필과 Supabase Auth 계정을 삭제한다. | 관리자 계정은 삭제할 수 없다. 관련 데이터는 정책에 따라 함께 정리된다. |
| ADM-05 | 도서 추가 | 관리자 | 제목, 저자, 출판 정보, 카테고리, 수량, 표지 및 설명을 입력해 도서를 등록한다. | 제목·저자·표지·1 이상의 전체 수량은 필수이다. |
| ADM-06 | 도서 자동 ID | 관리자 | 신규 도서 저장 시 도서 ID를 자동 생성한다. | 사용자가 직접 ID를 입력하지 않는다. |
| ADM-07 | 도서 수정 | 관리자 | 도서 목록의 수정 버튼을 누르면 팝업을 열어 기존 정보를 편집한다. | 현재 대출 수량보다 전체 수량을 작게 줄일 수 없다. |
| ADM-08 | 도서 삭제 | 관리자 | 확인 후 도서 데이터를 삭제한다. | 관리자 권한이 필요하다. 해당 도서의 대출 데이터는 외래키 정책에 따라 함께 삭제될 수 있다. |
| ADM-09 | 표지 이미지 선택 | 관리자 | JPG, JPEG, PNG, WEBP 파일을 선택하고 미리보기를 표시한다. | 최대 5MB만 허용한다. |
| ADM-10 | 표지 이미지 업로드 | 관리자 | 파일을 Supabase Storage의 공개 `book-covers` 버킷에 업로드하고 공개 URL을 도서 정보에 저장한다. | 업로드·수정·삭제는 관리자만 가능하고 열람은 공개된다. |
| ADM-11 | AI 정보 자동 작성 | 관리자 | 제목과 저자를 기준으로 출판사, 출판일, 카테고리, 키워드, 짧은 소개, 상세 설명의 빈 항목을 Gemini가 작성한다. | 관리자 인증 토큰과 Vercel의 `GEMINI_API_KEY`가 필요하다. 기존에 입력된 값은 덮어쓰지 않는다. |
| ADM-12 | AI 결과 검토 | 관리자 | 자동 작성 결과를 펼쳐서 확인하고 저장 전 직접 수정할 수 있다. | AI 결과는 사실과 다를 수 있으므로 최종 저장 전 관리자가 확인한다. |
| ADM-13 | 작성 중 데이터 보존 | 관리자 | 도서 추가 폼의 텍스트와 선택한 표지를 브라우저에 임시 저장하고 새로고침 후 복원한다. | 텍스트는 LocalStorage, 파일은 IndexedDB를 사용한다. 저장 성공 또는 입력 초기화 시 삭제한다. |

## 5. 핵심 업무 규칙

1. 신규 도서의 기본 전체 수량은 1권이다.
2. 도서의 전체 수량은 항상 1 이상이어야 한다.
3. 가용 수량은 0 이상이며 전체 수량을 초과할 수 없다.
4. 대출과 반납은 데이터베이스 함수에서 재고 변경과 대출 기록을 하나의 작업으로 처리한다.
5. 한 회원은 같은 도서를 동시에 두 번 대출할 수 없다.
6. 가용 수량이 0인 도서는 대출할 수 없다.
7. 본인이 대출 중인 도서에는 예약 버튼 대신 대여 중 상태를 표시한다.
8. 관리자는 도서를 대출하거나 예약할 수 없다.
9. 관리자 계정은 일반 회원 관리 목록에 표시하지 않으며 수정·삭제 대상에서 제외한다.
10. 회원 비밀번호는 암호화된 인증 시스템에서 관리하며 관리자도 원문을 볼 수 없다.

## 6. 주요 데이터 구성

| 데이터 | 저장 위치 | 주요 항목 |
|---|---|---|
| 인증 사용자 | Supabase Auth | 사용자 UUID, 이메일, 암호화된 인증정보, 가입일, 최근 로그인 |
| 회원 프로필 | `profiles` | 사용자 UUID, 로그인 아이디, 이름, 권한 |
| 도서 | `books` | 제목, 저자, 출판 정보, 카테고리, 키워드, 설명, 표지 URL, 전체·가용 수량 |
| 대출 | `book_loans` | 사용자, 도서, 상태, 대출일, 반납 예정일, 실제 반납일 |
| 표지 파일 | Supabase Storage `book-covers` | 관리자 업로드 이미지 파일 |
| 찜 | 브라우저 LocalStorage | 사용자 ID, 도서 ID, 생성일 |
| 예약 | 브라우저 LocalStorage | 사용자 ID, 도서 ID, 예약 상태, 생성일 |
| 최근 검색어 | 브라우저 LocalStorage | 최근 검색어 최대 5개 |
| 도서 작성 임시본 | LocalStorage·IndexedDB | 폼 입력값과 선택한 표지 파일 |

## 7. 보안 및 운영 조건

- Supabase publishable key만 브라우저에서 사용하고 service-role key와 Gemini API 키는 클라이언트 코드에 포함하지 않는다.
- Gemini API 키는 Vercel 환경변수에서만 읽는다.
- 데이터베이스 Row Level Security와 관리자 확인 함수로 회원·도서 관리 권한을 제한한다.
- 대출과 반납은 클라이언트에서 수량을 직접 수정하지 않고 인증된 데이터베이스 함수를 호출한다.
- 모든 사용자 입력은 화면 출력 전 HTML 이스케이프 처리를 적용한다.
- 표지 파일은 허용된 이미지 형식과 크기를 검사한다.
- Vercel 환경변수 변경 후에는 재배포해야 서버리스 함수에 적용된다.

## 8. 현재 구현 범위 및 향후 개선 사항

현재 대출과 도서 재고는 Supabase에 저장되어 계정 및 기기 간에 유지된다. 반면 찜과 예약은 브라우저 LocalStorage 기반이므로 다른 기기나 브라우저에서는 동기화되지 않고 브라우저 데이터를 삭제하면 사라질 수 있다. 실제 운영 수준으로 확장할 경우 찜과 예약도 Supabase 테이블과 서버 함수 기반으로 전환하고, 예약 순번·대출 한도·연체 처리·관리자 대출 현황 관리 기능을 추가할 수 있다.
